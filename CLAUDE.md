# 프로젝트 개요

비개발자가 Claude Code로 HTML 랜딩 페이지를 만들고, 배포/관리하는 프로젝트.
Cloudflare Workers에 GitHub 연동되어 있으며, main push 시 자동 빌드/배포된다.

- 배포 URL: `https://test.tunoinvest.com/{파일명}`
- 배포 대상: `deploy/` 폴더 안의 HTML 파일 (원본 파일명 그대로 배포)

# 폴더 구조

```
develop/    ← 작업 공간 (각자 개인 폴더에서 HTML 작업)
deploy/     ← 배포 공간 (사이트에 올라간 파일들)
archive/    ← 보관 공간 (사이트에서 내린 파일들)
guideline/  ← 가이드라인 모음 (HTML 생성 시 참고할 규칙/자료)
```

# 핵심 규칙

- 사용자는 **비개발자**다. git, 커밋, 푸시, 브랜치 같은 기술 용어를 쓰지 마라.
- "배포 완료", "업로드 완료" 같이 쉬운 말을 써라.
- 에러가 나면 기술적 메시지를 그대로 보여주지 말고, 무엇이 문제인지 쉽게 설명하라.
- 기본적으로 `develop/` 안의 개인 폴더에서 작업한다. 필요하면 `deploy/`, `archive/` 등 다른 폴더의 파일도 자유롭게 수정하고 배포할 수 있다.
- **같은 파일을 여러 사람이 동시에 수정하면 안 된다.** 충돌 시 개발팀에 문의하도록 안내하라.
- **deploy/ + archive/ 전체에서 파일명은 유니크해야 한다.**
- 배포 가능한 파일은 `.html`만이다.

# Git

- 브랜치는 main만 사용한다.
- push 전에 반드시 `git pull origin main`을 먼저 실행하라.
- 충돌 발생 시 사용자에게 "개발팀에 문의해주세요"라고 안내하고 중단하라.
- 커밋 메시지는 `.claude/conventions.md`를 따른다.

# HTML 생성 규칙

- 사용자가 HTML 생성을 요청하면, `guideline/` 폴더의 파일 목록을 먼저 보여줘라.
- 각 파일의 첫 줄(제목 또는 설명)을 함께 표시해서 사용자가 내용을 파악할 수 있게 하라.
- 사용자가 가이드라인을 선택하면 그 내용을 반영하되, 사용자의 요청사항도 함께 반영하라. 가이드라인을 선택하지 않아도 된다.
- HTML 저장 시 `git config user.name`으로 닉네임을 확인하고, `develop/{닉네임}/`에 저장하라. 닉네임에 해당하는 폴더가 없으면 사용자에게 `/처음설정`을 먼저 실행하라고 안내하라.
