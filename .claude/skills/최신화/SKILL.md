# 최신화

다른 사람이 올린 최신 내용을 내 컴퓨터에 받아온다.

## 실행 순서

아래 순서를 **정확히** 따르라. 중간에 실패하면 즉시 중단하고 무엇이 잘못됐는지 쉽게 설명하라.

### Step 1 — 최신 내용 받아오기

```bash
git pull origin main
```

- 성공하면 → 다음 단계로 진행.
- "Already up to date" → "이미 최신 상태예요!"라고 안내하고 종료.
- 충돌(conflict) 또는 로컬 변경사항과 겹쳐서 실패하면 → "내 작업과 다른 사람의 작업이 겹쳐요. 개발팀에 문의해주세요."라고 안내하고 중단하라.
- 네트워크 오류 등 기타 실패 → "인터넷 연결을 확인해주세요. 계속 안 되면 개발팀에 문의해주세요."라고 안내하고 중단하라.

### Step 2 — 결과 보고

받아온 변경사항을 간단히 요약한다.

```bash
git log --oneline HEAD@{1}..HEAD
```

위 명령어로 새로 받아온 커밋 목록을 확인하고, 사용자에게 쉽게 알려준다.

**보고 형식:**

```
최신화 완료! ✅

새로 받아온 내용:
- about.html 배포됨 (신규)
- test2.html 업데이트됨
- develop/userA/draft.html 수정됨

총 3개의 변경사항을 받아왔어요.
```

- 커밋 메시지의 prefix(`deploy:`, `archive:`, `save:`)를 보고 어떤 종류의 변경인지 쉬운 말로 설명하라.
- 커밋이 너무 많으면 (10개 이상) 최근 10개만 보여주고 "외 N개"로 요약하라.

## Example

**Input:**

```
/최신화
```

**결과 (새 내용 있음):**

```
최신화 완료! ✅

새로 받아온 내용:
- test1.html 배포됨 (신규)
- old-page.html 보관됨

총 2개의 변경사항을 받아왔어요.
```

**결과 (이미 최신):**

```
이미 최신 상태예요! ✅
```

**결과 (충돌):**

```
내 작업과 다른 사람의 작업이 겹쳐요.
개발팀에 문의해주세요.
```
