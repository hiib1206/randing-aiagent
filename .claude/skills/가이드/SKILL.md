# 프로젝트 규칙

이 프로젝트에서 지켜야 할 규칙을 안내한다. 사용자가 규칙을 물어보면 아래 내용을 쉬운 말로 알려줘라.

## 폴더 구조

```
develop/    ← 작업 공간 (여기서 HTML을 만든다)
deploy/     ← 배포 공간 (사이트에 올라간 파일들)
archive/    ← 보관 공간 (사이트에서 내린 파일들)
guideline/  ← 가이드라인 모음 (HTML 만들 때 참고할 규칙/자료)
```

## 규칙

### 1. 개인 작업 폴더를 기본으로 사용하라

- `develop/` 안에 각자의 개인 폴더가 있다. 기본적으로 여기서 작업하라.
- 필요하면 `deploy/`, `archive/` 등 다른 폴더의 파일도 자유롭게 수정하고 배포할 수 있다.
- **같은 파일을 여러 사람이 동시에 수정하면 안 된다.** 충돌이 발생하면 개발팀만 해결할 수 있다.

### 2. HTML 파일만 만들어라

- 이 프로젝트에서 배포 가능한 파일은 `.html`뿐이다.
- 다른 형식의 파일은 배포할 수 없다.

### 3. deploy/ + archive/ 파일명은 유니크해야 한다

- `deploy/`와 `archive/`를 합쳐서 같은 파일명이 존재하면 안 된다.
- 배포할 때 파일명이 그대로 사이트 주소가 된다. (예: `test.html` → `https://test.tunoinvest.com/test.html`)

### 4. 배포와 내리기는 스킬을 사용하라

- 처음 시작할 때는 `/처음설정`을 사용하라. (계정 설정, 작업 폴더 생성 등)
- 파일을 사이트에 올리고 싶으면 `/배포`를 사용하라. (신규 배포, 업데이트, 보관해제 모두 가능)
- 파일을 사이트에서 내리고 싶으면 `/보관`을 사용하라.
- 변경사항을 깃에 저장하고 싶으면 `/깃저장`을 사용하라.
- 이 규칙을 다시 보고 싶으면 `/가이드`를 사용하라.

### 5. HTML을 만들 때 가이드라인을 활용할 수 있다

- `guideline/` 폴더에 디자인 규칙, 브랜드 가이드 등 참고 자료가 있다.
- HTML을 만들어달라고 하면, 적용할 가이드라인을 선택할 수 있다.

## 명령어 사용법

명령어(`/배포` 등)를 직접 입력해도 되고, 자연스럽게 말로 해도 된다.

### `/처음설정` — 처음 시작할 때

처음 이 프로젝트를 사용하는 사람은 이 명령어를 먼저 실행하라.
계정 설정과 작업 폴더를 자동으로 만들어준다.

```
/처음설정
```

말로 할 때: "처음 설정", "처음설정 해줘"

### `/배포` — 파일을 사이트에 올리기

프로젝트 내 어디서든 HTML 파일을 배포할 수 있다.

```
/배포 파일명
```

말로 할 때: "{파일명}.html 배포해줘"

- 신규 파일이면 새로 배포된다.
- 이미 배포된 같은 파일이면 업데이트된다.
- 보관함에 있는 파일이면 다시 배포된다.
- 올라간 주소는 `https://test.tunoinvest.com/파일명` 형태로 알려준다.

### `/보관` — 파일을 사이트에서 내리기

사이트에 올린 페이지를 더 이상 보여주고 싶지 않을 때 사용한다.
파일은 삭제되지 않고 보관함에 보관된다.

```
/보관 파일명
```

말로 할 때: "{파일명} 내려줘", "{파일명} 배포 취소해줘", "{파일명} 페이지 내려줘", "{파일명} 보관 해줘"

### `/깃저장` — 변경사항 저장하기

프로젝트에서 변경한 내용을 깃에 저장한다.
배포와는 관계없이, 작업 내용을 안전하게 올려두고 싶을 때 사용한다.

```
/깃저장
```

말로 할 때: "깃에 저장해줘", "깃에 올려줘", "깃에 푸쉬해줘"

### `/가이드` — 이 안내 다시 보기

지금 보고 있는 이 규칙과 사용법을 다시 보고 싶을 때 사용한다.

```
/가이드
```

말로 할 때: "가이드", "규칙 알려줘", "주의사항", "뭐 하면 안 돼?", "이거 해도 돼?"
