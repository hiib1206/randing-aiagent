# 프로젝트 규칙

이 프로젝트에서 지켜야 할 규칙을 안내한다. 사용자가 규칙을 물어보면 아래 내용을 쉬운 말로 알려줘라.

## 폴더 구조

```
develop/    ← 작업 공간 (여기서 HTML을 만든다)
deploy/     ← 배포 공간 (직접 수정 금지, /deploy 스킬이 관리)
archive/    ← 보관 공간 (직접 수정 금지, 배포에서 내린 파일이 여기로 이동)
guideline/  ← 가이드라인 모음 (HTML 만들 때 참고할 규칙/자료)
```

## 규칙

### 1. 내 폴더에서만 작업하라 (가장 중요)

- `develop/` 안에 본인 폴더가 있다. 그 안에서만 작업하라.
- 다른 사람의 파일을 수정하지 마라.
- **같은 파일을 여러 사람이 동시에 수정하면 안 된다.** 충돌이 발생하면 개발팀만 해결할 수 있다.

### 2. HTML 파일만 만들어라

- 이 프로젝트에서 배포 가능한 파일은 `.html`뿐이다.
- 다른 형식의 파일은 배포할 수 없다.

### 3. deploy/, archive/ 폴더는 직접 건드리지 마라

- `deploy/`는 `/deploy` 스킬이 자동으로 관리한다.
- `archive/`는 `/archive` 스킬이 자동으로 관리한다. 배포에서 내린 파일이 여기로 이동된다.
- 이 두 폴더를 직접 수정하면 문제가 생길 수 있다.
- 단, 업데이트를 위해 `deploy/`에서 파일을 내 폴더로 복사해오는 것은 괜찮다.

### 4. 배포와 내리기는 스킬을 사용하라

- 처음 시작할 때는 `/setup`을 사용하라. (계정 설정, 작업 폴더 생성 등)
- 파일을 사이트에 올리고 싶으면 `/deploy`를 사용하라.
- 이미 올린 파일의 내용을 수정하고 싶으면 `/update`를 사용하라. (주소는 그대로, 내용만 바뀐다)
- 파일을 사이트에서 내리고 싶으면 `/archive`를 사용하라.
- 내렸던 파일을 다시 올리고 싶으면 `/restore`를 사용하라.
- 이 규칙을 다시 보고 싶으면 `/guide`를 사용하라.
- 직접 파일을 옮기거나 업로드하지 마라.

### 5. HTML을 만들 때 가이드라인을 활용할 수 있다

- `guideline/` 폴더에 디자인 규칙, 브랜드 가이드 등 참고 자료가 있다.
- HTML을 만들어달라고 하면, 적용할 가이드라인을 선택할 수 있다.

### 6. 배포하면 파일명이 자동으로 바뀐다

- 배포할 때 파일명이 v1.html, v2.html, v3.html... 처럼 자동 번호가 붙는다.
- 사이트에서 내려도 그 번호는 다시 사용되지 않는다 (v3을 내려도 다음 배포는 v4).

## 명령어 사용법

명령어(`/deploy` 등)를 직접 입력해도 되고, 자연스럽게 말로 해도 된다.

### `/setup` — 처음 시작할 때

처음 이 프로젝트를 사용하는 사람은 이 명령어를 먼저 실행하라.
계정 설정과 작업 폴더를 자동으로 만들어준다.

```
/setup
```

말로 할 때: "세팅해줘", "초기설정", "초기 세팅"

### `/deploy` — 파일을 사이트에 올리기

1. `develop/내폴더/` 안에 HTML 파일을 만든다.
2. 아래처럼 말하면 된다:

```
/deploy 파일명
```

말로 할 때: "{파일명}.html 배포해줘"

- 확인을 누르면 자동으로 사이트에 올라간다.
- 올라간 주소는 `https://test.tunoinvest.com/v번호` 형태로 알려준다.

### `/update` — 올린 파일 내용 수정하기

이미 사이트에 올린 페이지의 내용을 바꾸고 싶을 때 사용한다.
주소(v번호)는 그대로 유지되고, 내용만 바뀐다.

1. `deploy/` 폴더에서 수정할 파일을 `develop/내폴더/`로 복사한다.
2. 복사한 파일을 수정한다.
3. 아래처럼 말하면 된다:

```
/update {파일명}
```

말로 할 때: "{파일명} 업데이트해줘"

### `/archive` — 파일을 사이트에서 내리기

사이트에 올린 페이지를 더 이상 보여주고 싶지 않을 때 사용한다.
파일은 삭제되지 않고 보관함에 보관된다.

```
/archive {파일명}
```

말로 할 때: "{파일명} 내려줘", "{파일명} 배포 취소해줘", "{파일명} 페이지 내려줘"

### `/restore` — 내렸던 파일 다시 올리기

사이트에서 내렸던 페이지를 다시 올리고 싶을 때 사용한다.
원래 v번호 그대로 다시 올라간다.

```
/restore {파일명}
```

말로 할 때: "{파일명} 다시 올려줘", "{파일명} 다시 배포해줘", "{파일명} archive에서 올려줘"

### `/guide` — 이 안내 다시 보기

지금 보고 있는 이 규칙과 사용법을 다시 보고 싶을 때 사용한다.

```
/guide
```

말로 할 때: "가이드", "규칙 알려줘", "주의사항", "뭐 하면 안 돼?", "이거 해도 돼?"
